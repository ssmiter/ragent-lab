# Agent Loop 底层原理（简明版）

## 一图看懂

```
┌────────────────────────────────────────────────────────────────┐
│                        Agent Loop 执行流程                      │
├────────────────────────────────────────────────────────────────┤
│                                                                │
│  while (true) {                                                │
│      ┌──────────────────────────────────────────────────────┐  │
│      │ Turn N                                                 │  │
│      │                                                        │  │
│      │  ① turnCount++                    [0.1ms]            │  │
│      │  ② 检查护栏                         [0.1ms]            │  │
│      │  ③ 构建请求体                       [5ms]              │  │
│      │                                                        │  │
│      │  ④ ★ 调用 LLM API ★              [15-25秒]           │  │
│      │     └── 线程阻塞在这里，等待 HTTP 响应                │  │
│      │         这就是那 20 秒！                               │  │
│      │                                                        │  │
│      │  ⑤ 解析响应                         [1ms]              │  │
│      │  ⑥ 有 tool_calls?                                    │  │
│      │     ├── 有 → 执行工具                [1-5秒]           │  │
│      │     │       └── 也是 HTTP 调用，也会阻塞              │  │
│      │     └── 无 → 返回最终回答，退出循环                   │  │
│      │                                                        │  │
│      │  ⑦ 拼接消息历史                     [0.1ms]            │  │
│      └──────────────────────────────────────────────────────┘  │
│      │                                                         │
│      └──→ 继续 while 循环，进入 Turn N+1                       │
│  }                                                             │
│                                                                │
└────────────────────────────────────────────────────────────────┘
```

## 三个关键问题

### Q1: Turn 期间发生了什么？

```
Turn = 一次完整的"调模型 → 执行工具"循环

Turn 1: 调 LLM → 模型说"调工具 X" → 执行工具 X → 结果拼回历史
Turn 2: 调 LLM → 模型说"调工具 Y" → 执行工具 Y → 结果拼回历史
Turn 3: 调 LLM → 模型说"这是最终回答" → 返回用户
```

### Q2: while(true) 是一直跑吗？

```
不是！是"暂停-恢复"模式：

while (true) {
    快速操作 (毫秒)
    ↓
    ★ 网络请求 (秒级) ← 线程被挂起，等待响应
    ↓
    快速操作 (毫秒)
    ↓
    回到循环开头
}

CPU 使用率：接近 0%（因为 99% 时间在等网络）
```

### Q3: 20 秒在等什么？

```
等 HTTP 响应！

代码位置：
  String response = callBaiLianAPI(requestBody);
                   ↑
                   └── OkHttp 同步调用
                       client.newCall(request).execute()
                                                  ↑
                                                  └── 阻塞直到收到响应

这 20 秒期间：
  - main 线程状态: WAITING
  - while 循环: 停在这一行
  - CPU: 空闲
  - 网络: 等待数据包
```

## 时间都去哪了？

| 操作 | 耗时 | 说明 |
|-----|------|------|
| 循环控制 | 0.1ms | turnCount++、护栏检查 |
| 构建请求 | 5ms | 拼接 JSON |
| **等待 LLM** | **20s** | **主要耗时！** |
| 解析响应 | 1ms | 解析 JSON |
| 执行工具 | 2s | 也是网络请求 |
| **总计** | **~22s** | **99% 时间在网络等待** |

## 核心结论

1. **Agent Loop 是同步阻塞模型**
   - 代码简单，逻辑清晰
   - 适合串行依赖场景

2. **while(true) 不是"死循环"**
   - 每轮都会暂停等待网络
   - 不会占用 CPU

3. **瓶颈在网络，不在代码**
   - 20 秒是 LLM 推理时间
   - 优化方向：更快的模型、缓存、流式响应
