### 🔷 一、**顶层模块划分（Maven 多模块结构）**
项目根目录下划分为 **4 个核心子模块**，依赖关系明确：
```mermaid
graph LR
    bootstrap --> framework
    bootstrap --> infra-ai
    infra-ai --> framework
    mcp-server -.-> bootstrap & infra-ai
```

| 模块 | 定位 | 关键职责 |
|------|------|-----------|
| `framework` | **通用基础设施层** | 提供跨模块复用的基础能力：统一响应 `Result<T>`、异常体系（A/B/C 错误码）、用户上下文 `UserContext`（基于 `TransmittableThreadLocal`）、链路追踪 `RagTraceContext`、SSE 流式推送封装等。 |
| `infra-ai` | **AI 模型基础设施层** | 屏蔽大模型厂商差异，提供统一 AI 能力接口：<br>• `chat`：多模型对话路由（百炼/SiliconFlow/Ollama），支持流式/降级/熔断<br>• `embedding`：向量化服务（含路由与容错）<br>• `rerank`：重排序服务（如百炼重排模型）<br>• `model`：核心模型路由引擎（`ModelSelector` + `ModelRoutingExecutor` + `ModelHealthStore` 熔断器） |
| `bootstrap` | **业务主模块（入口）** | RAG+Agent 核心业务实现：<br>• `rag`：完整 RAG 流程（记忆加载 → 查询改写 → 意图识别 → 多通道检索 → 上下文组装 → 流式生成）<br>• `ingestion`：可配置文档 ETL 流水线（Fetcher→Parser→Chunker→Indexer）<br>• `knowledge`：知识库/文档/分块三级管理<br>• `admin`：Dashboard 运营看板<br>• `user`：Sa-Token 认证 |
| `mcp-server` | **独立 MCP 工具服务** | 实现 [MCP 协议](https://modelcontextprotocol.com/) 的 Spring Boot 应用：<br>• 接收 `POST /mcp` 请求，解析 JSON-RPC 2.0<br>• 动态注册工具（如 `WeatherMCPExecutor`, `TicketMCPExecutor`）<br>• 支持工具发现（`tools/list`）、调用（`tools/call`）和初始化 |

### 🔷 二、**关键分层与能力流（RAG+Agent 主流程）**
一次用户提问的端到端处理链路如下：

| 阶段 | 组件/模块 | 技术要点 |
|------|------------|----------|
| **输入预处理** | `bootstrap` + `knowledge` | • 关键词映射（规则替换）<br>• 查询改写（结合历史会话拆解/扩展问题）<br>• 意图识别（LLM 分类至意图树叶子节点：KB/系统回复/MCP） |
| **检索增强** | `bootstrap` (`rag`) + `infra-ai` | • 多通道并行检索：<br> ✓ `VectorGlobalSearchChannel`（全局 Milvus 向量检索）<br> ✓ `IntentDirectedSearchChannel`（按意图定向检索）<br>• 检索后处理：去重 → 重排序（`RerankPostProcessor`）→ 结果过滤 |
| **生成与交互** | `bootstrap` (`rag`) + `infra-ai` (`chat`) | • `RAGPromptService` 组装 Prompt（含检索内容 + 系统指令）<br>• 调用 `LLMService` 流式生成 → 通过 `SseEmitterSender` 推送前端<br>• 支持中断取消（`StreamCancellationHandle`） |
| **工具调用（Agent）** | `mcp-server` ↔ `bootstrap` | • 当意图为 `MCP` 类型时，`bootstrap` 向 `mcp-server` 发起 HTTP 调用<br>• `mcp-server` 解析请求 → 查找注册工具 → 执行 → 返回结构化结果 → 交由 LLM 组织自然语言回答 |

### 🔷 三、**支撑性基础设施**
- **存储层**：
    - MySQL（业务元数据：知识库/文档/用户）
    - Milvus（向量存储，用于语义检索）
    - Redis（会话缓存、分布式锁、意图树缓存）
    - RustFS（对象存储，Milvus 索引文件 + Ragent 文档原文件）
- **可观测性**：
    - 全链路追踪（`RagTraceContext` + `@RagTraceRoot/@RagTraceNode` AOP）
    - Dashboard 运营看板（成功率/响应时间/无知识率等实时指标）
- **部署架构**：
    - Docker Compose 管理中间件（Milvus/RustFS/Redis/MySQL）
    - 前端：React 18 + TypeScript + Vite
    - 后端：Spring Boot 3.5.7 + Java 17
