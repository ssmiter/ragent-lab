# Experiment 2：多工具编排

## 实验目标

验证 Agent Loop 的"按需重写"是否优于 Pipeline 的"无脑总是重写"。

## 核心假设

> 模型能自主判断：清晰问题直接搜，模糊问题先重写再搜。

## 实现文件

| 文件 | 说明 |
|-----|------|
| `query_rewrite_investigation.md` | ragent 重写能力调查 |
| `QueryRewriteMCPExecutor.java` | 查询重写工具（mcp-server） |
| `AgentLoopExperiment2.java` | 实验入口（双工具） |

---

## 测试结果汇总

| 问题 | 组别 | 是否重写 | 搜索次数 | 总轮次 | 结果 |
|-----|------|---------|---------|--------|------|
| Q1 | A | ❌ 否 | 1 | 2 | ✅ 高分直接回答 |
| Q7 | B | ✅ 是（2次） | 2 | 5 | ⚠️ 低分重试后诚实告知 |

## 详细分析

### Q1（A组）— 清晰问题直接搜索 ✅

**原始问题**：`Ragent系统的整体架构是什么？`

**模型思考**（Turn 1 前）：
```
问题清晰、具体，直接使用关键词"Ragent系统 整体架构"进行知识库检索。
```

**工具调用链**：
```
[Turn 1] knowledge_search_with_rerank({"query": "Ragent系统 整体架构"}) -> ✓
[Turn 2] 模型生成回答
```

**关键发现**：
- ✅ 模型正确判断"问题清晰"，**跳过重写**
- ✅ 路径A（直接搜索）被正确选择
- ✅ 2 轮完成，效率高

---

### Q7（B组）— 模糊问题多次重写 ⚠️

**原始问题**：`之前看过一个什么自适应的东西，好像和分数有关？`

**工具调用链**：
```
[Turn 1] rewrite_query({"original_query": "..."}) -> ✓
  → 改写: "自适应截断策略 分数阈值"
[Turn 2] knowledge_search_with_rerank({"query": "自适应截断策略 分数阈值"}) -> ✓
  → 分数: 0.53（低）
[Turn 3] rewrite_query({"original_query": "...", "intent_hint": "教育测评领域..."}) -> ✓
  → 改写: "教育测评 自适应测试 分数估计 能力评估"
[Turn 4] knowledge_search_with_rerank({"query": "..."}) -> ✓
  → 分数: 0.55（仍低）
[Turn 5] 模型诚实告知知识库无匹配，但给出专业解释
```

**关键发现**：
- ✅ 模型正确判断"问题模糊"，**主动重写**
- ✅ 模型使用了 `intent_hint` 参数引导重写方向
- ✅ 低分时**自适应重试**：换角度重新改写
- ✅ 最终诚实告知：知识库无相关内容，但提供专业解释

**模型思考亮点**（Turn 3）：
```
所有检索结果的 Rerank 分数均为 0.53，远低于 0.75 的可信阈值...
更合理的重写方向应聚焦高频术语组合：
→ "自适应测试 能力分数"
→ "CAT 项目反应理论 分数"
```

---

### Q10（B组）— 口语化问题完美重写 ✅

**原始问题**：`那个把文档切成小块的功能咋实现的`

**工具调用链**：

[Turn 1] rewrite_query({"original_query": "..."}) -> ✓
  → 改写: "文档分块策略实现"
[Turn 2] knowledge_search_with_rerank({"query": "文档分块策略实现"}) -> ✓

**关键发现**：
- ✅ 口语化 → 技术术语，改写质量高
- ✅ 3 轮完成，效率高

## 核心结论

### 1. 模型的编排判断力 ✅ 验证通过

| 场景 | 模型选择 | 是否正确 |
|-----|---------|---------|
| 清晰技术问题（Q1） | 路径A：直接搜索 | ✅ |
| 模糊口语问题（Q10） | 路径B：先重写再搜索 | ✅ |
| 低分结果（Q7） | 路径C：重试 + 换角度 | ✅ |

### 2. 重写的投入产出比

| 问题 | 重写价值 | 说明 |
|-----|---------|------|
| Q1 | 无需重写 | 问题已精确，重写是浪费 |
| Q10 | 高价值 | 口语→术语，检索质量显著提升 |
| Q7 | 有价值但无匹配 | 重写方向正确，知识库缺内容 |

### 3. 超预期的模型能力

1. **自适应重试策略**：Q7 中模型主动判断低分 → 换角度重写
2. **参数使用**：模型学会了使用 `intent_hint` 引导重写
3. **诚实回答**：知识库无匹配时，不胡编乱造

### 4. Agent Loop vs Pipeline

| 维度 | Agent Loop | Pipeline |
|-----|-----------|----------|
| Q1（清晰问题） | 2轮，跳过重写 ✅ | 固定重写，浪费1次LLM调用 |
| Q10（口语问题） | 3轮，重写后高分 ✅ | 重写后检索，效果相当 |
| Q7（无匹配问题） | 5轮，多角度尝试 ⚠️ | 固定流程，1次重写就放弃 |

**结论**：Agent Loop 在清晰问题上**更高效**，在困难问题上**更健壮**。
