# CC Task 007：前端接入智能路由 + 部署验证

> **目标：** 让前端默认走 `/smart/chat` 智能路由端点，用户无需手动选择模式。部署后验证路由准确性。
> **约束：** 前端最小改动。Nginx 配置由操作者处理，CC 不需要管。
> **产出：** `CC_Report_007_Smart_Frontend.md`

---

## 背景

Task 006 已完成 Strategy + Router 架构，`/smart/chat` 端点可用。但前端仍在用旧逻辑（手动切换 Pipeline/Agent 端点）。本任务将前端接入智能路由，完成闭环。

---

## 任务内容

### 1. 前端改造

调查前端当前的模式切换逻辑（`agentModeEnabled` 状态和端点切换），然后做最小改动：

**核心目标：** 新增"智能模式"作为默认选项，请求走 `/smart/chat`。

具体方案由你根据前端实际代码决定，以下是几种可能的思路（选最简单的）：

- 方案 A：将模式切换改为三选一（Pipeline / Agent / 智能），智能为默认
- 方案 B：去掉模式切换，统一走智能路由，保留 URL 参数 `?strategy=xxx` 作为调试后门
- 方案 C：其他你认为更合理的方式

**原则：** 用户打开页面就是智能模式，不需要任何操作就能用。想手动指定模式的高级用户仍然有办法切换。

### 2. 部署验证

部署后用实际请求验证（操作者会提前配好 Nginx）：

- 简单问题是否路由到 Pipeline（观察日志中的策略选择记录）
- 复杂问题是否路由到 Agent
- SSE 流式返回是否正常
- 对话历史是否正常工作
- 原有端点 `/rag/v3/chat` 和 `/agent/chat` 是否仍可用

> 如果验证中发现 bug，授权直接修复，在 REPORT 中说明。
