## 调研步骤

### 1. 从 app1.log 找到这次请求的完整日志

在 app1.log 中搜索关键词"介绍一下你自己"或"SYSTEM"，
找到对应的一次完整请求记录，提取：
- 请求进入哪个 Controller / 端点
- 意图识别结果是什么（日志中的原始输出）
- 何时触发了 Agent 模式转交
- Agent Loop 执行了几轮、调用了哪些工具

### 2. 追踪 SYSTEM 意图的定义和识别逻辑

找到意图识别相关代码，回答：
- SYSTEM 意图是在哪里定义的（配置文件/数据库/代码中的枚举？）
- 大模型意图识别的 Prompt 模板是什么
- 识别到 SYSTEM 意图后，转交 Agent 的触发点在哪个类/方法

### 3. 追踪 Agent 接管后的 Prompt 组装

找到 Agent 模式下 Prompt 组装的代码，回答：
- 系统 Prompt 内容是什么（或从哪里读取）
- 工具描述如何注入
- 用户原始问题如何拼接
- 最终发给大模型的完整 Prompt 结构是什么样的

### 4. 追踪 system_info_query 工具的调用

找到该工具的实现，回答：
- 工具返回的是哪些真实数据（知识库列表从哪里查询）
- Agent 调用工具后如何把结果拼回上下文
